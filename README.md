# RepoScout

RepoScout - это CLI-агент, который помогает быстро выбрать GitHub-репозитории для изучения новой технической темы.

Я делал его под ситуацию, когда человек примерно понимает область, но не знает, с какого проекта начать. Например: "AI agents", "computer vision for beginners", "math modeling python". Агент не отвечает из головы: он вызывает GitHub API, получает реальные репозитории, ранжирует их и объясняет выбор.

## Что умеет агент

- принимает обычный текстовый запрос;
- формирует поисковый запрос для GitHub;
- вызывает внешний API GitHub Search;
- сохраняет найденные данные в `data/*.json`;
- ранжирует репозитории по простым и объяснимым признакам;
- формирует итоговый ответ на русском языке.

## Инструменты агента

1. `search_github_repositories` - ходит в GitHub Search API и получает реальные репозитории.
2. `rank_repositories` - локально оценивает проекты по звездам, свежести, описанию, темам и признакам проекта для новичка.

Это закрывает два трека:

- **Track A**: агент использует внешний API.
- **Track C**: агент получает данные, приводит их к структуре JSON и отвечает с опорой на эти данные.

## Быстрый запуск

Нужен Python 3.9+.

Перед первым запуском установите зависимости через pip:

```bash
python3 -m pip install -r requirements.txt
```

Запуск интерактивного режима:

```bash
python3 main.py
```

Внутри можно написать обычный запрос:

```text
> Найди GitHub-проекты про AI agents для новичка
```

Можно запустить и одним запросом без интерактивного режима:

```bash
python3 main.py "Найди GitHub-проекты про AI agents для новичка"
```

Для полноценного LLM-режима нужно добавить ключ. Без ключа можно запустить запасной режим, где LLM не вызывается, но GitHub API и ранжирование работают:

```bash
python3 main.py --offline "Найди GitHub-проекты про AI agents для новичка"
```

## Команды CLI

В интерактивном режиме доступны команды:

```text
/help                 показать список команд
/tools                показать инструменты агента
/skills               показать skill-режимы
/skill on <name>      включить skill
/skill off <name>     выключить skill
/exit                 выйти
```

Пример:

```text
> /skills
> /skill off beginner_mode
> Найди проекты по computer vision для новичка
```

## Настройка LLM

Скопируйте пример env-файла:

```bash
cp .env.example .env
```

Заполните:

```bash
LLM_API_KEY=ваш_ключ
LLM_BASE_URL=https://api.agentrouter.org/v1
LLM_MODEL=claude-haiku-4-5-20251001
```

Если в Agent Router в личном кабинете указан другой Base URL, нужно поставить его в `LLM_BASE_URL`. Код использует OpenAI-compatible endpoint `/chat/completions`.

GitHub token необязателен, но с ним выше лимиты:

```bash
GITHUB_TOKEN=github_pat_...
```

## Как это работает

1. Агент получает запрос пользователя.
2. LLM-планировщик решает, нужно ли искать репозитории.
3. Если поиск нужен, вызывается инструмент `search_github_repositories`.
4. Результат сохраняется в `data/generated/` вместе с датой сбора и источником.
5. Инструмент `rank_repositories` считает прозрачную оценку.
6. LLM или запасной шаблон собирает финальный ответ.

Если LLM недоступна, агент все равно может работать в запасном режиме: он сам соберет поисковый запрос для GitHub, вызовет API и покажет список. Для сдачи я использую LLM-режим, потому что по условию нужен AI-агент, а не только скрипт.

## Источники данных

- GitHub Search API: `https://api.github.com/search/repositories`
- Пример структуры данных лежит в `data/example_search_result.json`.
- Новые результаты запусков сохраняются в `data/generated/`.
- Дата сбора пишется в каждый JSON-файл в поле `collected_at`.

## Skill и tool

Tool - это функция, которую агент реально вызывает для действия или получения данных. В этом проекте tools лежат в `tools/`: поиск через GitHub API и ранжирование.

Skill - это режим поведения агента, то есть дополнительная инструкция. Skill сам ничего не делает, а меняет стиль и правила ответа. В этом проекте skills лежат в `skills/`: `beginner_mode` и `strict_sources`.

## Ограничения

- Звезды GitHub не равны качеству проекта.
- GitHub Search иногда возвращает популярные, но слишком сложные репозитории.
- Без GitHub token можно быстрее упереться в лимит запросов.
- Агент пока не читает весь код репозитория, поэтому оценивает проект по публичным метаданным.
